이메일 검증 인증서의 경우:
ACM 인증서가 있는 AWS 리소스를인터넷의 HTTPS 요청을 수락합니다.를 선택합니다. 인증서의 도메인 이름에 대한 HTTPS 요청이 인증서가 들어 있는 리소스로 라우팅되는지 확인합니다.

DNS 검증 인증서의 경우:
DNS 구성의 CNAME 레코드가 올바른지 확인합니다.

이메일 검증 인증서에 대한 관리형 인증서 갱신
이메일 검증 인증서는 825일마다 도메인을 검증해야 합니다. 갱신을 진행하기 위해 ACM은 도메인 이름에 대한 이메일을 전송합니다.PENDING_VALIDATION상태로 유지됩니다. 도메인 소유자나 도메인의 공인 대리인은 검증에 실패한 각 도메인 이름을 검증하는 작업을 수행해야 합니다. 섹션을 참조하십시오.이메일을 통한 확인에 있는 도메인을 식별하는 방법에 대한 지침은PENDING_VALIDATION상태를 유지하고 해당 도메인에 대한 유효성 검사 프로세스를 반복합니다.

DNS 검증 인증서에 대한 관리형 인증서 갱신
ACM은 DNS 검증 인증서에 대한 TLS 검증을 시도하지 않습니다. ACM이 DNS 검증으로 검증한 인증서를 갱신하지 못한다면, 가장 가능성이 높은 원인은 DNS 구성에 CNAME 레코드가 없거나 정확하지 않기 때문입니다. 이런 상황이라면 ACM에서는 인증서를 자동으로 갱신할 수 없다고 알립니다. 올바른 CNAME 레코드를 DNS 데이터베이스에 삽입해야 합니다. ACM 콘솔에서 인증서와 도메인 항목을 확장하면 도메인에 대한 CNAME 레코드를 찾을 수 있습니다. 자세한 내용은 아래 그림을 참조하십시오. 사용 하 여 CNAME 레코드를 검색할 수도 있습니다.DescribeCertificate작업 또는 ACM API에서인증서 설명명령을 ACM CLI에서 실행할 수 있습니다. 자세한 내용은 옵션 1: DNS 검증 단원을 참조하세요.
